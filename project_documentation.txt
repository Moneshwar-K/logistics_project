1. PROJECT OVERVIEW

Project Title: LogisticHub - Integrated Logistics & ERP Solutions

Problem Statement:
Traditional logistics and freight forwarding operations often rely on fragmented systems, manual data entry, and siloed communication. This leads to high error rates in billing, lack of real-time visibility for customers, and operational bottlenecks in branch management and shipment dispatching.

Objective of the Project:
To develop a comprehensive, end-to-end ERP solution that digitizes the entire logistics lifecycle—from HAWB (House Airway Bill) booking and automated invoice generation to manifest creation, real-time tracking, and branch-level inventory management.

Applications:

- International & Domestic Courier Services

2. SYSTEM DESCRIPTION

Overall Architecture:
The system follows a modern Full-Stack Decoupled Architecture:
- Frontend: Single Page Application (SPA) built with Next.js for high performance and SEO readiness.
- Backend: RESTful API built with Node.js and Express.js, handling business logic and security.
- Database: MongoDB (NoSQL) for flexible and scalable data storage.
- Authentication: JWT (JSON Web Tokens) for stateless, role-based access control.

Tech Stack:
- Frontend: Next.js 14, React.js, TailwindCSS, Lucide Icons.
- Backend: Node.js, Express.js, Mongoose ODM.
- Database: MongoDB.
- Tools: TypeScript, Axios, JWT, Postman (Testing), Git.

Workflow: How the system works:
1. User Management: Admins set up branches, service types, and staff accounts.
2. HAWB Booking: Operations staff enter shipment details (Hawb, weight, dimensions, shipper/consignee).
3. Billing: The system automatically calculates rates and generates Invoices and Duty Bills based on predefined rate cards.
4. Dispatch: Shipments are grouped into Manifests for transit and eventually assigned to DRS (Delivery Run Sheets) for final delivery.
5. Customer Portal: Customers log in to view their specific shipments, download invoices, and track parcel status in real-time.


3. MODULE DESCRIPTION

Frontend Module:
- Purpose: Provides an interactive interface for staff and customers.
- Functionality: Features nested routing for admin, billing, and operations sections. Includes complex forms with focus management and real-time validation.
- Tools: Next.js, TailwindCSS.

Backend Module:
- Purpose: Manages data persistence, security, and business logic.
- Functionality: Implements over 20+ API endpoints for CRUD operations on shipments, users, and financials. Includes middleware for JWT validation and role-based authorization.
- Tools: Express.js, Node.js.

Database Module:
- Purpose: Stores structured logistics data.
- Functionality: Designs relationships between Branches, Users, Parties (Clients), and Shipments using Mongoose Schemas.
- Tools: MongoDB.


4. FUNCTIONAL REQUIREMENTS

- Authentication: Dual login portals (Staff/Customer) with distinct dashboards.
- Shipment Management: Create, update, and search HAWBs.
- Financials: Automated Invoice and Duty Bill generation with PDF printing support.
- Logistics Operations: Manifest creation, DRS generation, and status updates (In-Transit, Delivered, etc.).
- Search & Filtering: Advanced filtering for shipments by date, branch, and status.
- Master Data: Management of Branches, Employees, Clients, and Rate Cards.


5. NON-FUNCTIONAL REQUIREMENTS

- Performance: Optimized database queries and frontend rendering for fast load times.
- Security: Password hashing (Bcrypt), JWT protection, and data isolation by role.
- Scalability: NoSQL architecture allows for handling increasing shipment volumes.
- Usability: Clean, professional UI/UX designed for high-speed data entry.

7. WORK SUMMARY

Implementation Details:
- Developed a robust authentication system allowing role-based access to specific ERP modules.
- Refactored frontend input components to resolve character-by-character rendering bugs.
- Integrated automated invoice calculation logic based on volumetric weight and rate slabs.
- Unified the operational flow from booking to delivery run sheets.

Challenges Faced:
- Managing complex state transitions in multi-step billing forms.
- Ensuring data integrity across multiple linked collections (HAWB -> Invoice -> Manifest).
- Resolving focus-loss issues in React components during high-frequency typing.

Solutions:
- Extracted nested form components to prevent unnecessary DOM re-mounting.
- Utilized Mongoose middleware for automated sequence generation (e.g., Bill numbers).
- Implemented centralized error handling in the Express backend.
